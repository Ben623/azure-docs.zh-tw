# <a name="what-is-blockchain-data-manager-for-azure-blockchain-service"></a>什麼是適用于 Azure 區塊鏈 Service 的區塊鏈資料管理員？

區塊鏈資料管理員會在 azure 事件方格主題中捕獲、轉換及傳遞 Azure 區塊鏈服務交易資料，以提供可靠且可擴充的區塊鏈總帳與 Azure 服務的整合。

在大部分的企業區塊鏈案例中，區塊鏈總帳是解決方案的一個部分。 例如，若要將資產從一個實體轉移到另一個，您需要一個機制來提交交易。 接著，您需要一種讀取總帳資料的機制，以確保交易發生、已接受，而且產生的狀態變更會與您的端對端解決方案整合。 在此範例中，如果您撰寫智慧合約來傳送資產，您可以使用區塊鏈資料管理員來整合離鏈應用程式和資料存放區。 針對資產傳輸範例，當資產在區塊鏈上傳輸時，事件和屬性狀態變更會由區塊鏈資料管理員透過事件方格來傳遞。 接著，您可以使用事件方格的多個可能事件處理常式，將區塊鏈資料儲存在外，或即時回應狀態變更。

區塊鏈資料管理員會執行三個主要功能： [捕捉]、[轉換] 和 [傳遞]。

![區塊鏈資料管理員函式](./media/data-manager/functions.png)

## <a name="capture"></a>擷取

每個區塊鏈資料管理員實例會連接到一個 Azure 區塊鏈 Service 成員交易節點。 只有具有交易節點存取權的使用者可以建立連線，以確保對客戶資料的存取控制正確。 區塊鏈資料管理員實例會從交易節點可靠地捕捉所有原始區塊和未經處理的交易資料，並可進行調整以支援企業工作負載。

## <a name="transform"></a>轉換

您可以藉由在區塊鏈資料管理員中設定智慧型合約應用程式，使用區塊鏈資料管理員來解碼事件和屬性狀態。 若要新增智慧合約，您需提供合約 ABI 和位元組碼。 區塊鏈資料管理員會使用智慧合約成品來解碼和探索合約位址。 將區塊鏈應用程式新增至實例之後，區塊鏈資料管理員會在智慧合約部署至聯盟並將已解碼的事件和屬性狀態傳送至設定的目的地時，動態地探索智慧合約位址。

## <a name="deliver"></a>傳遞

區塊鏈資料管理員針對任何指定的區塊鏈資料管理員實例支援多個事件方格主題輸出連接。 您可以將區塊鏈資料傳送至單一目的地，或將區塊鏈資料傳送至多個目的地。 使用區塊鏈資料管理員，您可以為任何區塊鏈部署建立可擴充的事件型資料發佈解決方案。

## <a name="configuration-options"></a>組態選項

您可以設定區塊鏈資料管理員，以符合解決方案的需求。 例如，您可以布建：

* 適用于 Azure 區塊鏈服務成員的單一區塊鏈資料管理員實例。
* 每個 Azure 區塊鏈服務交易節點的區塊鏈資料管理員實例。 例如，私用交易節點可以有自己的區塊鏈資料管理員實例，以維護機密性。
* 區塊鏈資料管理員實例可以支援多個輸出連接。 一個區塊鏈資料管理員實例可以用來管理 Azure 區塊鏈服務成員的所有資料發佈整合點。
