## <a name="azure-maps-supported-views"></a>Azure 地圖服務支援檢視

> [!Note]
> 我們即將於 2019 年 8 月 1 日，在下列國家/地區中發行 Azure 地圖服務：
>  * 阿根廷
>  * 印度
>  * 摩洛哥
>  * 巴基斯坦
>
> 在 2019 年 8 月 1 日之後**檢視**參數設定會定義新的區域/國家/地區上面所列的傳回的對應內容。 我們鼓勵您確定已設定 View 參數所需的 REST Api 和 Sdk，使用您的服務。
>  
>
>  **Rest Api:**
>  
>  請確定您已設定視需要檢視參數。 View 參數會指定透過 Azure Maps 服務傳回的一組的地理政治有爭議的內容。 
>
>  受影響的 Azure 地圖服務 REST 服務：
>    
>    * 取得地圖底圖
>    * 取得地圖影像 
>    * 取得搜尋模糊
>    * 取得搜尋 POI
>    * 取得搜尋的 POI 分類
>    * 取得搜尋附近
>    * 取得搜尋地址
>    * 取得搜尋結構化的地址
>    * 取得搜尋服務位址的反向
>    * 取得搜尋地址反向交叉街道
>    * 幾何內的 post 搜尋
>    * Post 搜尋地址批次預覽
>    * Post 搜尋地址反向的批次預覽
>    * Post 搜尋路由
>    * Post 搜尋模糊的批次預覽
>
>    
>  **Sdk:**
>
>  確定您已設定為必要項，View 參數，且您有 Web SDK 和 Android SDK 的最新版本。 受影響的 Sdk:
>
>    * Azure 地圖服務 Web SDK
>    * Azure 地圖服務 Android SDK


Azure 地圖服務**檢視**參數 （也稱為 「 使用者區域參數 」） 是兩個字母 ISO 3166 國家/地區程式碼會顯示正確的對應，針對指定的地理政治集該國家/地區有爭議的內容傳回透過 Azure Maps 服務，包括框線和地圖上顯示的標籤。 

View 參數設定為預設**整合**，即使您未在要求中所定義。 您必須負責判斷您的使用者的位置，然後設定 View 參數，該位置的正確。 或者，您可以選擇設定 ' 檢視 = Auto'，這會傳回要求的 IP 位址為基礎的地圖資料。  在 Azure 地圖服務檢視參數必須使用符合當地法律，包括與國家/地區位置地圖、 影像和其他資料和第三方內容，您會授權存取透過 Azure 地圖服務的相關對應可供。
