# <a name="multiclass-decision-forest-module"></a>多元決策樹系的模組

本文說明 Azure Machine Learning 服務的視覺化介面 （預覽） 的模組。

使用此模組來建立機器學習模型，根據*決策樹系*演算法。 決策樹系是快速建置一系列的學習加上標籤的資料時的決策樹集團模型。

## <a name="more-about-decision-forests"></a>深入了解決策樹系

決策樹演算法是集團學習方法來分類。 此演算法的運作方式是建立多個決策樹，然後*投票*最受歡迎的輸出類別上。 投票是一種彙總的每個樹狀目錄中分類決策樹系輸出非標準化的頻率長條圖的標籤。 彙總程序會加總這些長條圖，並正規化為 「 機率 」 取得的每個標籤的結果。 有很高的預測信心的樹狀結構中的集團最終批覆有更大的權數。

決策樹大致上非參數化的模型，這表示它們支援各種分佈的資料。 每個樹狀結構中，在一系列簡單的測試執行的每個類別，增加樹狀結構的層級，分葉節點 （決策） 為止。

決策樹有許多優點：

+ 它們可以代表非線性決策界限。
+ 它們是在運算和記憶體使用效率定型和預測期間。
+ 執行整合式的特徵選取和分類。
+ 也就是彈性呈現雜訊特徵。

在 Azure Machine Learning 中的決策樹系分類器是由一整團的決策樹所組成。 一般而言，集團模型會提供更佳的涵蓋範圍和精確度比單一決策樹。 如需詳細資訊，請參閱 <<c0> [ 決策樹](https://go.microsoft.com/fwlink/?LinkId=403677)。

## <a name="how-to-configure-multiclass-decision-forest"></a>如何設定多級決策樹系



1. 新增**多級決策樹系**模組至您在介面中的實驗。 您可以找到此模組下的**Machine Learning**，**初始化模型**，並**分類**。

2. 按兩下以開啟模組**屬性**窗格。

3. 針對**重新取樣方法**，選擇用來建立個別的樹狀結構的方法。  您可以選擇封袋或複寫。

    + **封袋**:也稱為封袋*啟動程序的彙總*。 在此方法中，每個樹狀結構成長上新的範例中，建立隨機取樣以取代原始資料集，除非您有一個資料集的原始大小。 藉由組合模型的輸出*投票*，這是一種彙總。 如需詳細資訊，請參閱維基百科項目，用來啟動程序的彙總。

    + **複寫**:在複寫中，每個樹狀結構是完全相同的輸入資料上定型。 決定哪個分割的述詞用於每個樹狀節點會保留隨機的建立不同的樹狀結構。

   

4. 指定您要模型定型，藉由設定的方式**建立定型模式**選項。

    + **單一參數**:如果您知道您要設定模型，並提供一組值做為引數的方式，請選取此選項。


5. **決策樹數目**:輸入可在集團中建立的決策樹的最大數目。 藉由建立多個決策樹，您或許能夠較佳的涵蓋範圍，但是定型時間可能會增加。

    此值也會控制視覺化定型的模型時，在結果中，顯示的樹狀結構數目。 若要查看或列印單一樹狀結構，您可以設定的值為 1。不過，這表示該只有一個樹狀結構可能會產生 （樹狀目錄中的使用一組初始參數），而且不會執行任何進一步的反覆項目。

6. **決策樹的最大深度**:輸入數字，以限制任何決策樹的最大深度。 增加樹狀結構的深度可能會提高精確度，風險是有過度配適和拉長定型時間。

7. **每個節點的隨機分割數目**:輸入建置樹狀結構的每個節點時使用的分割數目。 A*分割*表示樹狀結構 （節點） 的每個層級中的功能會隨機分割。

8. **每個分葉節點範例的最小數目**:表示在樹狀目錄中建立任何終端節點 （分葉） 所需的情況下最小數目。 藉由增加此值，您會增加建立新規則的臨界值。

    例如，預設值為 1，即使單一案例可能會造成要建立新的規則。 如果您增加到 5 的值時，定型資料必須包含至少五個符合相同條件的案例。



10. 加上標籤的資料集，以及其中一個定型模組連線：

    + 如果您設定**建立定型模式**要**單一參數**，使用[定型模型](./train-model.md)模組。

11. 執行實驗。

## <a name="results"></a>結果

訓練完成之後：

+ 若要查看樹狀目錄中建立的每個反覆項目，以滑鼠右鍵按一下的輸出[定型模型](./train-model.md)模組，然後選取**視覺化**。
+ 若要查看每個節點的規則，請按一下向下切入到分割的每個樹狀結構。
