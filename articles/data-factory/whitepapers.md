# <a name="azure-data-factory-whitepapers"></a>Azure Data Factory 技術白皮書 

技術白皮書可讓您瀏覽更深層級的 Azure Data Factory。 本文章提供您一份可用的白皮書針對 Azure Data Factory。

| **白皮書** | **說明** |
| --- | --- |
|[Azure Data Factory，在雲端中的資料整合](https://azure.microsoft.com/mediahandler/files/resourcefiles/azure-data-factory-data-integration-in-the-cloud/Azure_Data_Factory_Data_Integration_in_the_Cloud.pdf) | 本文件說明 Azure Data Factory 如何讓您建置現代化資料倉儲，請啟用進階的分析，以驅動智慧型的 SaaS 應用程式，您 SQL Server 整合服務將套件提取至 Azure。|
|[從內部部署關聯式資料倉儲的資料移轉至 Azure 中使用 Azure Data Factory](https://azure.microsoft.com/mediahandler/files/resourcefiles/data-migration-from-on-premise-relational-data-warehouse-to-azure-data-lake-using-azure-data-factory/Data_migration_from_on-prem_RDW_to_ADLS_using_ADF.pdf) | 此白皮書討論移轉的複雜度數十 TB 資料從現有的內部部署關聯式資料倉儲 (例如 Netezza、 Oracle、 Teradata、 SQL server) 至 Azure （例如，Blob 儲存體或 Azure Data Lake 儲存體） 使用 Azure Data Factory。 周圍恢復能力、 效能、 延展性、 管理和安全性至 Azure 的 Azure Data Factory 的巨量資料擷取旅程圖說明的挑戰和最佳作法。 |
|[Azure Data Factory:在雲端中的 SSIS](https://azure.microsoft.com/mediahandler/files/resourcefiles/azure-data-factory-ssis-in-the-cloud/Azure_Data_Factory_SSIS_in_the_Cloud.pdf)| 您要將您現有的 SSIS 工作負載移轉至 Azure Data Factory 和位址一般考量和問題的原因超過這份文件。 然後，我們將逐步引導您完成建立 Azure SSIS IR 的技術詳細資料，然後示範如何上傳、 執行和監視您的封裝，透過 Azure Data Factory 可讓您使用的工具可能已熟悉像是 SQL Server Management Studio (SSMS)。 |
|[Azure data Factory – 傳遞參數](https://azure.microsoft.com/mediahandler/files/resourcefiles/azure-data-factory-passing-parameters/Azure%20data%20Factory-Whitepaper-PassingParameters.pdf)| 這篇文章會引導您如何將參數傳遞的管線及活動內的活動之間。 |
|[Azure data Factory-DevOps](https://azure.microsoft.com/mediahandler/files/resourcefiles/whitepaper-adf-on-azuredevops/Azure%20data%20Factory-Whitepaper-DevOps.pdf)| 本白皮書涵蓋一些持續整合的最佳作法和部署 Azure Data Factory。 |
