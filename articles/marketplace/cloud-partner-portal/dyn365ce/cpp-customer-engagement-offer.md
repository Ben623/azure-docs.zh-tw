# <a name="dynamics-365-for-customer-engagement-offer"></a>Dynamics 365 for Customer Engagement 供應項目

|     |     |
| --- | --- |
| <div style="font-size:1em"> 本節說明如何將 Dynamics 365 for Customer Engagement 解決方案發佈到 Microsoft [AppSource Marketplace](https://appsource.microsoft.com)。 Dynamics 365 for Customer Engagement 的所有應用程式 (Sales、Service、Project Service 及 Field Service) 都必須經過我們的認證程序並支援試用體驗。 認證程序會檢查您的解決方案是否符合標準需求、相容性和適當的做法。 試用體驗可讓使用者將解決方案部署至即時 Dynamics 365 環境。 </div>|  ![Dynamics 365 Customer Engagement 圖示](./media/dynce-icon1.png)  |


## <a name="publishing-overview"></a>發佈概觀

[在 Microsoft AppSource (組建 2018) 上提供吸引人的體驗](https://www.youtube.com/watch?v=WWBFuNHC0J4)這部影片提供 AppSource Marketplace 的概觀，以及此生態系統的優點。 接著，還示範自訂的 Dynamics 365 for Customer Engagement 應用程式範例、說明提供「試用產品」試用的優點，以及如何發佈到 AppSource。

> [!Video https://channel9.msdn.com/events/Build/2018/THR1202/player]


## <a name="publishing-process"></a>發佈程序

本節中的後續文章將引導您完成建立和發佈 Dynamics 365 for Customer Engagement 應用程式的程序。  建議您參閱 Dynamics 365 文件的[在 AppSource 上發佈應用程式](https://docs.microsoft.com/dynamics365/customer-engagement/developer/publish-app-appsource)一節中對應的資訊。


## <a name="next-steps"></a>後續步驟

建立新的 Dynamics 365 for Customer Engagement 供應項目之前，您必須符合此供應項目類型的[必要條件](./cpp-prerequisites.md)。
