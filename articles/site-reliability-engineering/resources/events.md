# <a name="participate-in-the-global-site-reliability-engineering-community"></a>參與全球網站可靠性工程社區

就像在 SRE 上閱讀一樣重要的是，與您的對等溝通通常會比較重要。 討論您在 SRE 方面的挑戰、成功和失敗，對於取得主旨的差別細微瞭解十分重要。

有一些功能是 SRE 內容的聚會和會議。 最直接相關的是全球散發的[SREcon 會議](https://www.usenix.org/srecoon)，USENIX。

有越來越多的 SRE 內容正在進行像是[LISA](https://www.usenix.org/conferences/byname/5)、[全天 DevOps](https://www.alldaydevops.com)、 [DevOps Enterprise 峰](https://events.itrevolution.com)和當地 DevOps 會議（例如[DevOps Days](https://www.devopsdays.org)）的會議。 找出此內容，以及其他對主旨感興趣的地方。
