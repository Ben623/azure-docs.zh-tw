# <a name="what-is-devops"></a>什麼是 DevOps？
DevOps （「開發」和「作業」的組合）強調軟體發展人員與生產 IT 專業人員之間的溝通與共同作業的重要性，同時自動部署軟體和基礎結構變更。 它著重于採用 agile 和精簡方法，並將重複性工作自動化，並以即時、成功的應用程式版本為目標。

Microsoft 透過[Azure DevOps](https://azure.microsoft.com/services/devops/) （雲端）和[Visual Studio App Center](https://azure.microsoft.com/services/app-center/)提供了一組完整的[DevOps 解決方案](https://azure.microsoft.com/solutions/devops/)。 這些產品所建立的工作環境，可以快速、頻繁且可靠地進行軟體的建立、測試和部署。 它們也可讓組織更快達到其目標，以在部署新功能、安全性修補程式和 bug 修正時提供更快速的周轉時間。

**DevOps 的重要性**
- 從程式碼認可到生產部署，讓整個軟體生命週期更快。
- 可讓組織在內部及客戶更快速地提供軟體和安全性更新。
- 旨在減少開發人員專注于滿足商務需求來加入功能及改善應用程式的可用性時所發生的衝突，但營運小組在傳統上是著重于計算的穩定性和安全性環境.
- 提供更高的靈活性、改善客戶回應能力，並縮短上市時間。
- 將人員、程式和產品整合在一起，以持續傳遞價值給終端使用者，以擴充靈活性。
- DevOps 實務會降低成本，而且所有組織都能更有效地達成其目標，不論其任務為何。
