# <a name="what-is-azure-service-health"></a>什麼是 Azure 服務健康狀態？

Azure 提供一套的體驗，以讓您隨時掌握您的雲端資源的健全狀況。 這項資訊包括目前和即將發行的問題，例如服務影響事件、 計劃性的維護和其他變更可能會影響您的可用性。

Azure 服務健康狀態是由三個個別較小的服務組成。

[Azure 狀態](azure-status-overview.md)上會通知您在 Azure 中的服務中斷 **[Azure 狀態頁面](https://status.azure.com)** 。 分頁是在所有 Azure 區域的所有 Azure 服務的健康情況的全域檢視。 [狀態] 頁面是不錯的參考，事件具有廣泛的影響，但強烈建議您目前的 Azure 使用者運用，隨時掌握 Azure 的事件和維護相關的 Azure 服務健康狀態。

[Azure 服務健康狀態](service-health-overview.md)提供個人化的檢視，您使用的區域與 Azure 服務的健全狀況。 由於受驗證的 Azure 服務健康狀態體驗知道您目前使用的服務及資源，因此您可以在此尋找各項受影響服務的通訊，包括服務中斷、計劃性維護活動以及其健康情況諮詢。 您可以設定服務健康狀態警示，以充分利用服務健康狀態的功能，並透過慣用的通訊管道收到關於服務問題、計劃性維護或其他可能影響 Azure 服務和使用區域之變更的相關資訊。

[Azure 資源健康狀態](resource-health-overview.md)提供您的個別的雲端資源，例如特定的虛擬機器執行個體的健康情況資訊。 使用 Azure 監視器，您也可以設定警示，以便在雲端資源可用性變更時收到通知。 Azure 資源健康狀態搭配 Azure 監視器通知會協助您及時收到資源可用性的相關通知，並快速評估問題根源是出自於您，或與 Azure 平台事件相關。

這些經驗值加在一起，會提供您更全面細微且最相關的 Azure 健康情況檢視。
