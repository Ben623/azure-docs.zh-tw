# <a name="access-and-use-workspaces-on-the-my-apps-preview-portal"></a>在我的應用程式 (預覽) 入口網站上存取和使用工作區

工作區是您在 [我的應用程式]  (預覽) 入口網站中看到的各種應用程式檢視。 例如，應用程式可以依部門功能或使用者角色分組到工作區。 如果您未看到任何可用的工作區或類別，這表示您的系統管理員尚未設定任何工作區或類別或與您共用它們。 請洽詢組織的技術服務人員，以取得其他協助或權限來查看共用的工作區。

[!INCLUDE [preview-notice](../../../includes/active-directory-end-user-my-apps-and-workspaces.md)]

>[!Important]
>本內容適用於使用者。 如果您是系統管理員，可以在[應用程式管理文件](https://docs.microsoft.com/azure/active-directory/manage-apps/access-panel-workspaces)中找到更多關於如何設定和管理雲端式應用程式的資訊。

## <a name="access-apps-using-workspaces"></a>使用工作區存取應用程式

[我的應用程式]  入口網站中的工作區清單預設會顯示一個名為**所有應用程式**的工作區，其中包含您有權存取的每個應用程式：

![[我的應用程式] 入口網站中的 [所有應用程式] 頁面](media/my-applications-portal-workspaces/my-apps-all.png)

您在此清單中看到的任何其他工作區，都是由您的系統管理員所建立並與您共用。 您可以選擇這其中一個工作區來查看範圍更廣泛的應用程式集，例如下列範例中的「支援」  ：

![[我的應用程式] 入口網站中個別的工作區和已設定範圍的應用程式](media/my-applications-portal-workspaces/my-apps-workspace.png)

當您選取並檢視工作區時，只會看到您有權存取的預設應用程式。

## <a name="next-steps"></a>後續步驟

在將應用程式組織為 [我的應用程式]  入口網站中的各種類別之後，您可以：

- 檢閱、更新或撤銷已授與應用程式的權限。 如需詳細資訊，請參閱[在我的應用程式 (預覽) 入口網站中編輯或撤銷應用程式權限](my-applications-portal-permissions-saved-accounts.md)。
