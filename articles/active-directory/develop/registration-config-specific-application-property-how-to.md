# <a name="how-to-fill-out-specific-fields-for-a-custom-developed-application"></a>如何針對自訂開發的應用程式填寫特定欄位

本文提供 [Azure 入口網站](https://portal.azure.com)上應用程式註冊表單中所有可用欄位的簡短說明。

## <a name="register-a-new-application"></a>註冊新的應用程式

-   若要註冊新的應用程式，請瀏覽至 [Azure 入口網站](https://portal.azure.com)。

-   從左側瀏覽窗格中，按一下 [Azure Active Directory]  。

-   選擇 [應用程式註冊]  ，然後按一下 [新增]  。

-   這樣會開啟應用程式註冊表單。

## <a name="fields-in-the-application-registration-form"></a>應用程式註冊表單中的欄位


| 欄位            | 描述                                                                              |
|------------------|------------------------------------------------------------------------------------------|
| 名稱             | 應用程式的名稱。 至少應該有四個字元。                |
| 支援的帳戶類型| 選取哪些帳戶，您想要讓應用程式支援： 僅此組織目錄中的帳戶、 帳戶在任何組織的目錄中，或任何組織的目錄中的帳戶和個人的 Microsoft 帳戶。  |
| 重新導向 URI （選用） | 選取您要建置的應用程式類型**Web**或是**公開用戶端 （行動和桌面）** ，然後輸入您的應用程式的 重新導向 URI （或回覆 URL）。 若為 Web 應用程式，請提供應用程式的基底 URL。 例如， http://localhost:31544 可能是在您的本機電腦上執行之 Web 應用程式的 URL。 使用者會使用此 URL 來登入 Web 用戶端應用程式。 若為公用用戶端應用程式，請提供 Azure AD 用來傳回權杖回應的 URI。 您的應用程式，例如 myapp://auth 輸入特定的值。若要查看 Web 應用程式或原生應用程式的特定範例，請參閱我們的[快速入門](https://docs.microsoft.com/azure/active-directory/develop)。|

一旦您填妥上述欄位，在 Azure 入口網站中註冊應用程式，並將您重新導向至應用程式的 [概觀] 頁面。 在左窗格中的 [設定] 頁面**管理**有更多欄位供您自訂您的應用程式。 下表描述的所有欄位。 您只會看到這些欄位，取決於您建立 web 應用程式或公用用戶端應用程式的子集。

### <a name="overview"></a>總覽
| 欄位           | 描述        |
|-----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 應用程式識別碼  | 當您註冊應用程式時，Azure AD 會將應用程式識別碼指派給您的應用程式。 應用程式識別碼可在對 Azure AD 的驗證要求中用來唯一識別您的應用程式，還可用來存取資源 (例如圖形 API)。                                                          |
| 應用程式識別碼 URI      | 這應該是唯一的 URI，通常形式為 **https://&lt;tenant\_name&gt;/&lt;application\_name&gt;。** 這用在授權授與流程中，為指定的資源，應該會發行權杖的唯一識別碼。 它在核發的存取權杖中也會變成 'aud' 宣告。 |

### <a name="branding"></a>商標

| 欄位           | 描述        |
|-----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 上傳新標誌 | 您可以使用此欄位上傳應用程式的標誌。 標誌必須是.bmp、.jpg 或.png 格式，而且檔案大小應該小於 100 KB。 影像尺寸應該是 215 x 215 像素，而中央影像尺寸為 94 x 94 像素。|
| 首頁 URL   | 這是在應用程式註冊期間指定的登入 URL。|

### <a name="authentication"></a>驗證

| 欄位           | 描述        |
|-----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 登出 URL      | 這是單一登出的登出 URL。 當使用者透過 Azure AD 使用任何其他已註冊的應用程式來清除他們的工作階段時，Azure AD 會傳送登出要求給這個 URL。|
| 支援的帳戶類型  | 這個參數指定應用程式是否可供多重租用戶使用。 一般而言，這表示外部組織可以在其租用戶中註冊應用程式，並授權存取其組織的資料，以使用您的應用程式。|
| 重新導向 Url      | 重新導向或回覆，Url 會是 Azure AD 傳回您的應用程式要求之任何權杖的所在端點。 針對原生應用程式，會在成功授權之後將使用者傳送到此 URI。 Azure AD 會檢查重新導向您的應用程式在 OAuth 2.0 要求中所提供的 URI 符合其中一個入口網站中已註冊的值。|

### <a name="certificates-and-secrets"></a>憑證和祕密

| 欄位           | 描述        |
|-----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 用戶端祕密            | 祕密或金鑰，以程式設計方式存取的網站沒有任何使用者互動的 Azure AD 所保護的 Api，您可以建立用戶端。 從**新的用戶端祕密**頁面上，輸入金鑰描述和到期日，然後儲存產生的金鑰。 務必將它儲存在安全的位置，因為您稍後將無法存取它。             |
