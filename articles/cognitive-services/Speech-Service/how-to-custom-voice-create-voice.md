# <a name="create-a-custom-voice"></a>建立自訂的語音

在 [準備資料以自訂語音](how-to-custom-voice-prepare-data.md)，我們會說明您可用來訓練自訂的語音和不同的格式需求的不同資料類型。 一旦您已備妥您的資料，就可以開始上傳到[自訂語音入口網站](https://aka.ms/custom-voice-portal)，或透過自訂語音訓練 API。 我們在這裡說明的訓練自訂的語音，透過入口網站的步驟。

> [!NOTE]
> 本頁面假設您已閱讀[開始使用自訂語音](how-to-custom-voice.md)並[準備資料以自訂語音](how-to-custom-voice-prepare-data.md)，而且已建立的自訂語音專案。

檢查支援的自訂語音的語言：[語言自訂](language-support.md#customization)。

## <a name="upload-your-datasets"></a>上傳您的資料集

當您準備要上傳您的資料時，請前往[自訂語音入口網站](https://aka.ms/custom-voice-portal)。 建立或選取自訂語音專案。 專案必須共用正確的語言/地區和 gender 屬性做為資料要用於您的語氣訓練課程。 例如，選取`en-GB`如果您有音訊錄製是以英文使用英國腔調字。

移至**資料**索引標籤，然後按一下**將資料上傳**。 在精靈中，選取符合您已備妥正確的資料類型。

您上傳每個資料集必須符合您所選擇的資料類型的需求。 請務必正確地格式化您的資料，才能上傳。 這可確保自訂語音服務將會正確處理資料。 移至[準備資料以自訂語音](how-to-custom-voice-prepare-data.md)，並確定已正確地訂定格式化您的資料。

> [!NOTE]
> 免費訂用帳戶 (F0) 使用者可以同時上傳兩個資料集。 標準訂用帳戶 (S0) 使用者可以同時上傳 5 個資料集。 若達到限制，請等候直到至少其中一個資料集完成匯入。 然後再試一次。

> [!NOTE]
> 要匯入每個訂用帳戶允許的資料集的數目上限是 10 個.zip 檔案免費訂用帳戶 (F0) 使用者和標準訂用帳戶 (S0) 使用者的 500。

一旦您按 [上傳] 按鈕，會自動驗證資料集。 資料驗證包含一系列的檢查以確認其檔案格式、 大小和取樣率的音訊檔案。 如果有的話，請修正錯誤，並再次提交。 已成功起始匯入資料的要求時，您應該會看到對應至您剛才上傳的資料集的資料表中的項目。

下表顯示所匯入資料集的處理狀態：

| State | 意義 |
| ----- | ------- |
| Processing | 您的資料集已收到，正在處理中。 |
| Succeeded | 您的資料集已驗證，而且現在可用來建立語音模型。 |
| Failed | 您的資料集已失敗，因為許多原因，例如檔案錯誤，資料的問題或網路問題而處理期間。 |

完成驗證之後，您可以看到在您的資料集的每個相符的談話總數**發音**資料行。 如果您已選取的資料類型需要長時間音訊分割，則此資料行只會反映談話，我們已分割，根據您的成績單或透過語音轉譯服務。 您可以進一步下載驗證，以檢視詳細資料的結果已成功匯入的談話，而且其對應的文字記錄的資料集。 提示： 長時間音訊分割可能需要超過一小時才能完成資料處理。

對於 EN-US 」 和 「 ZH-CN 的資料集，您可以進一步下載報表以檢查每個錄製的發音分數和雜訊程度。 發音分數介於 0 到 100 的範圍間。 70 以下的分數通常表示語音錯誤或腳本不符。 大量腔調字可能會降低您的發音分數，並影響產生的數位語音。

信噪比 (SNR) 愈高表示您音訊中的雜音愈低。 在專業錄音室中，通常可達到 50+ SNR。 SNR 低於 20 的音訊可能會在您產生的語音中導致明顯的雜音。

請考慮以較低的發音分數或較差的信噪比來重新錄製任何語句。 若無法重新錄製，您可以從資料集中排除這些語句。

## <a name="build-your-custom-voice-model"></a>建置您的自訂語音模型

已驗證您的資料集之後，您可以使用它來建置您的自訂語音模型。

1.  瀏覽至**文字轉換語音 > 自訂語音 > 訓練**。

2.  按一下 **定型模型**。

3.  接下來，輸入**名稱**並**描述**可協助您識別此模型。

    請謹慎選擇名稱。 您在這裡輸入的名稱，會是您在語音合成要求中指定語音作為 SSML 輸入一部分所使用的名稱。 字母、 數字和少數的標點符號字元，例如- \_，和 （'、'） 允許。 使用不同的語音模型不同的名稱。

    [描述]  欄位的常見用法是記錄建立模型時所用的資料集名稱。

4.  從**選取定型資料**頁面上，選擇您想要用於訓練的一或多個資料集。 您送出之前，請檢查談話的數目。 您可以與任意數目的 EN-US 」 和 「 ZH-CN 語音模型的表達方式開始。 其他地區設定中，您必須選取超過 2,000 個無法訓練語音的談話。

    > [!NOTE]
    > 從訓練，將會移除重複的音訊名稱。 請確定您選取的資料集不包含相同的音訊名稱跨多個.zip 檔案。

    > [!TIP]
    > 需要高品質的結果使用相同的演講者備忘資料集。 當您提交的訓練資料集包含總數小於 6000 的相異談話時，您將定型您的語音模型，透過參數化的合成統計技術。 在案例中將訓練資料超過 6,000 相異的談話總數的位置，您將會開始以串連合成的技術訓練程序。 通常串連技術可能會導致更自然，和精確性更高的語音的結果。 [請連絡小組自訂語音](mailto:speechsupport@microsoft.com)如果您想要定型模型，以使用最新的類神經 TTS 技術可能會產生相當於可公開取得的數位語音[類神經語音](language-support.md#neural-voices)。

5.  按一下 **定型**開始建立您的語音模型。

定型資料表會顯示新的項目對應到這個新建立的模型。 資料表也會顯示狀態：處理、 成功、 失敗。

顯示的狀態會反映將您的資料集轉換成語音模型的程序，如下所示。

| State | 意義 |
| ----- | ------- |
| Processing | 正在建立您的語音模型。 |
| Succeeded | 您的語音模型已建立，而且可以部署。 |
| Failed | 您的語音模型已訓練，因為許多原因，例如未察覺的資料的問題或網路問題而失敗。 |

訓練時間會因所處理的音訊資料量而有所不同。 一般時間範圍為數百個語句 30 分鐘到 20,000 個語句 40 小時。 一旦成功訓練模型，您可以開始測試它。

> [!NOTE]
> 免費訂用帳戶 (F0) 使用者可以同時定型一個製化聲音音調。 標準訂用帳戶 (S0) 使用者可以同時訓練三個語音。 若達到限制，請等候直到至少其中一個聲音音調完成訓練，然後再試一次。

> [!NOTE]
> 語音模型進行定型每訂用帳戶允許的數目上限是 10 個免費訂用帳戶 (F0) 使用者的模型和標準訂用帳戶 (S0) 使用者的 100。

## <a name="test-your-voice-model"></a>測試您的語音模型

在成功建立聲音音調後，您就可以進行測試，再將它部署以供使用。

1.  瀏覽至**文字轉換語音 > 自訂語音 > 測試**。

2.  按一下 **新增測試**。

3.  選取您想要測試的一或多個模型。

4.  提供您想要讀出 voice(s) 的文字。 如果您已選取一次測試多個模型，相同的文字將用於不同模型的測試。

    > [!NOTE]
    > 您的文字語言必須與聲音音調語言相同。 您可以測試只有成功定型的模型。 在此步驟中支援僅純文字。

5.  按一下頁面底部的 [新增]  。

一旦您已經提交您的測試要求，您會回到 [測試] 頁面。 資料表中現在包含與新的要求對應的項目，以及狀態資料行。 合成語音可能需要幾分鐘的時間。 當 [狀態] 欄寫著**Succeeded**，您可以播放音訊，或下載的文字輸入 （.txt 檔），以及音訊輸出 （.wav 檔案），並進一步聆聽後者的品質。

您也可以找到您所選取每個模型的詳細資料頁面中的測試結果進行測試。 移至**訓練**索引標籤，然後按一下 模型名稱，輸入模型詳細資料頁面。

## <a name="create-and-use-a-custom-voice-endpoint"></a>建立並使用自訂語音端點

成功建立並測試語言模型之後，您可以將它部署至自訂文字轉換語音端點。 然後，您可以使用此端點取代透過 REST API 提出文字轉換語音要求時的慣用端點。 只能由您用來部署之字型的訂用帳戶，就可以呼叫您的自訂端點。

若要建立新的自訂語音端點，請前往**文字轉換語音 > 自訂語音 > 部署**。 選取**新增端點**，然後輸入**名稱**並**描述**適用於您的自訂端點。 然後，選取您想要與這個端點關聯的自訂語音模型。

您已按下後**新增** 按鈕，在 端點 資料表中，您會看到一個項目為新的端點。 具現化新的端點可能需要幾分鐘的時間。 當部署狀態為 [成功]  時，端點就可供使用。

> [!NOTE]
> 免費訂用帳戶 (F0) 使用者可以部署只能有一個模型。 標準訂用帳戶 (S0) 使用者可以建立最多可到 50 個端點，每個都有它自己自訂的語音。

> [!NOTE]
> 若要使用您自訂的語音，您必須指定語音模型名稱、 直接在 HTTP 要求中使用自訂的 URI 以及通過 TTS 服務的驗證中使用相同的訂用帳戶。

部署您的端點之後，將端點名稱會顯示為連結。 按一下連結以顯示您的端點，例如端點索引鍵、 端點 URL 和範例程式碼的特定資訊。

端點的線上測試也可透過自訂語音入口網站使用。 若要測試您的端點，請選擇**檢查端點**從**端點詳細資料**頁面。 [Endpoint Testing] \(端點測試\) 頁面隨即出現。 輸入要讀出的文字 (以純文字或[SSML 格式](speech-synthesis-markup.md)在文字方塊中。 若要聆聽以您自訂聲音音調說出的文字，請選取 [播放]  。 針對您的客製化語音合成使用量支付此測試的功能。

自訂端點的功能與用於文字轉換語音要求的標準端點完全相同。 如需詳細資訊，請參閱 [REST API](rest-text-to-speech.md)。
