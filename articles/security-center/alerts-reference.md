# <a name="security-alerts---a-reference-guide"></a>安全性警示-參考指南

本文列出您可能會在 Azure 資訊安全中心的威脅防護模組中看到的安全性警示。 您的環境中顯示的警示取決於您要保護的資源和服務，以及您的自訂設定。

若要瞭解如何回應這些警示，請參閱[管理和回應 Azure 資訊安全中心中的安全性警示](security-center-managing-and-responding-alerts.md)。

若要瞭解如何匯出警示（和建議），請參閱[匯出安全性警示和建議（預覽）](continuous-export.md)。

在 [警示] 資料表下方是一個資料表，描述用來分類這些警示意圖的 Azure 資訊安全中心終止鏈。

## <a name="intentions"></a>意向

瞭解攻擊的目的可協助您更輕鬆地調查和報告事件。 Azure 資訊安全中心警示包括 [意圖] 欄位，以協助進行這些工作。

描述從偵察網路攻擊至資料外泄之進度的一系列步驟，通常稱為「終止鏈」。 

資訊安全中心支援的終止鏈意圖是根據[MITRE ATT & CK™架構](https://attack.mitre.org/matrices/enterprise/eh)，並在下表中說明。

|Intent|說明|
|------|-------|
|**探查**|探查可能會嘗試存取特定的資源，而不受惡意意圖的限制，或嘗試在攻擊之前取得目標系統的存取權，以收集資訊。 此步驟通常會被偵測為嘗試（源自于網路外部）來掃描目標系統並識別進入點。|
|**時會**|入侵是攻擊者管理以取得受攻擊資源據點的階段。 此階段與計算主機和資源相關，例如使用者帳戶、憑證等。威脅執行者通常可以在此階段之後控制資源。|
|**持續性**|持續性是系統的任何存取、動作或設定變更，可讓威脅執行者在該系統上持續存在。 威脅動作專案通常需要透過中斷（例如系統重新開機、認證遺失或其他可能需要遠端存取工具來重新開機的失敗）來維護系統的存取權，或提供替代的後門程式讓他們重新取得存取權。|
|**PrivilegeEscalation**|許可權提升是動作的結果，可讓敵人在系統或網路上取得較高層級的許可權。 某些工具或動作需要較高層級的許可權，才能正常運作，而且在整個作業的許多點上都有可能需要。 具有許可權可存取特定系統，或執行敵人以達成其目標所需特定功能的使用者帳戶，也可能會被視為提升許可權。|
|**DefenseEvasion**|防禦規避包含敵人可能用來規避偵測或避免其他防禦的技巧。 有時候，這些動作與其他類別中的（或各種不同的）技術相同，這些都具有破壞特定防禦或緩和措施的額外優點。|
|**CredentialAccess**|認證存取代表的技術可讓您存取或控制在企業環境中使用的系統、網域或服務認證。 敵人可能會嘗試從使用者或系統管理員帳戶（具有系統管理員存取權的「本機系統管理員」或「網域使用者」）取得合法的認證，以在網路中使用。 有了網路內的足夠存取權，敵人就可以建立帳戶供稍後在環境中使用。|
|**探索|探索包含的技術可讓敵人取得系統和內部網路的相關知識。 當敵人取得新系統的存取權時，他們必須將自己導向至他們現在可以控制的內容，以及從該系統運作的優點，在入侵期間提供給其目前目標或整體目標。 作業系統提供許多原生工具，可協助此入侵後的資訊收集階段。|
|**LateralMovement**|橫向移動包含的技術可讓敵人存取和控制網路上的遠端系統，但不一定會包含在遠端系統上執行工具。 橫向移動技術可讓敵人從系統收集資訊，而不需要額外的工具，例如遠端存取工具。 敵人可以使用橫向移動來進行許多用途，包括遠端執行工具、切換到其他系統、存取特定資訊或檔案、存取其他認證，或造成影響。|
|**執行**|執行策略代表會導致在本機或遠端系統上執行受敵人控制之程式碼的技術。 這種策略通常與橫向移動搭配使用，以擴充網路上遠端系統的存取權。|
|**集合**|集合包含的技術，可在外泄之前，用來識別並收集來自目標網路的資訊，例如機密檔案。 此類別也涵蓋系統或網路上敵人可能尋找資訊以竊取的位置。|
|**外洩**|外泄是指在敵人從目標網路中移除檔案和資訊時，所產生或協助的技術和屬性。 此類別也涵蓋系統或網路上敵人可能尋找資訊以竊取的位置。|
|**CommandAndControl**|命令和控制策略代表敵人如何與目標網路內控制之下的系統進行通訊。|
|**影響**|影響事件主要會嘗試直接減少系統、服務或網路的可用性或完整性;包括資料的操作，以影響商務或操作程式。 這通常指的是勒索軟體、受損、資料操作和其他技術。|
